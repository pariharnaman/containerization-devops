#  Experiment 5: Docker - Volumes, Environment Variables, Monitoring & Networks


## Step 1: Understanding Data Persistence

### The Problem: Container Data is Ephemeral

- Data inside a container is temporary  
- When the container stops or is removed, data is lost  
- Containers do not store data permanently  

➡️ We need a way to persist data outside the container  


![Container Ephemeral Data](./Images/cont.png)

![Container Ephemeral Data](./Images/ephemeral.png)

## Step 2: Volume Types

Docker provides different types of volumes for data persistence:

### a) Named Volumes
- Managed by Docker  
- Easy to use and reusable  
- Stored in Docker’s default location  
![Volume Types](./Images/namedvol.png)

### b) Anonymous Volumes
- Created automatically  
- No specific name assigned  
- Hard to manage and reuse  
![Volume Types](./Images/anonymousvol.png)

### c) Bind Mounts
- Links a container to a specific path on the host  
- Full control over files  
- Useful for development  

![Volume Types](./Images/bindmount.png)
